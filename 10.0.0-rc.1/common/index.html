Common SDK

Library maintainer: @mapbox/core-sdk

Description

This library provides native and platform-specific common functionality among all Mapbox SDKs via Mapbox Bindgen IDL files. These files are individual per service. We use Bindgen in order to generate bindings for all supported platforms: Linux, Android, and iOS.

The following APIs are available:

  • Accounts API
  • Billing Service API
  • Configuration Service API
  • Events Service API
  • HTTP Service API
  • Location Service API
  • Logging API
  • Network Connectivity API
  • Reachability API
  • System Information API
  • Tile Store API
  • Value API
  • Version API

Another goal of the Common SDK is to harmonize data types we use across SDKs. In order to do that, the common library provides:

  • Shared data types (enforced via bindgen primitive types)
  • Pinned versions of Mapbox Base and Mapbox Bindgen

Getting Started

C++ SDK developers can integrate with the Common SDK by including it as a git submodule to allow building from source and to get debugging capabilities.

Developers can choose to link statically or dynamically but they should bear in mind that linking statically is only intended for enterprise customers, internal customers (e.g. SDK devs) should link dynamically.

When SDK developers ship their products, they should not merge the Common SDK into their product. Doing this would at best only increase the binary size of the SDK but it also runs the risk of introducing clashing symbols. Instead, Common SDK is distributed separately so e.g Navigation and Maps can link with the same library. As mentioned, we strongly recommend linking against the Common SDK dynamically when building your SDKs for distribution. See DEVELOPMENT#Release for more information.
